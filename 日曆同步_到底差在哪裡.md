# 日曆同步到底差在哪裡？

一句話：**程式與流程都接好了，只差「Supabase 裡 3 個 Secrets 填對」＋「Google 日曆有分享給服務帳號」。**  
目前實測卡在 **私鑰格式**，下面用對照表說明。

---

## 流程對照：有做 vs 還差什麼

| 步驟 | 程式／你已做的 | 還差／容易錯的 |
|------|----------------|----------------|
| 1. 客人下單 | ✅ 預約寫入 DB，api-booking 會呼叫 google-calendar-sync | 無 |
| 2. 取得 Google 存取權 | ✅ Edge Function 用私鑰簽 JWT、換 token | ❌ **GOOGLE_SERVICE_ACCOUNT_PRIVATE_KEY** 在 Supabase 裡格式錯誤 → 出現「Failed to decode base64」 |
| 3. 在日曆建立活動 | ✅ 呼叫 Google Calendar API 建立事件 | ❌ 若日曆沒分享給服務帳號 → 403 |
| 4. 寫回預約 | ✅ 把 `google_calendar_event_id` 寫回 bookings | 上一步成功就會有 |

所以「差在哪裡」= **第 2 步（私鑰）** 或 **第 3 步（日曆分享）**，多數是第 2 步。

---

## 第 2 步：私鑰到底差在哪裡？

- **程式要的**：能還原成 PEM 的一串字（含 `-----BEGIN PRIVATE KEY-----`、中間 base64、`-----END PRIVATE KEY-----`），中間換行可以是 `\n` 或實際換行。
- **Supabase Secrets 的限制**：你貼上一大段多行文字時，有時會被截斷、或換行被吃掉、或多了/少了引號，導致**不是「完整一段 PEM」**，解 base64 就失敗 → 日曆同步永遠卡在「Failed to decode base64」。

**正確做法（建議一步一步做）：**

1. 打開你下載的**服務帳號 JSON 檔**（例如 `xxx.json`）。
2. 找到 **`"private_key"`** 這個欄位，值是一串用雙引號包起來、裡面有 `\n` 的文字。
3. **整段複製**：從欄位值開頭的第一個 `"` 到結尾的 `"`（**包含雙引號**），中間不要刪、不要手動換行。
4. 到 **Supabase** → **Edge Functions** → **google-calendar-sync** → **Settings** → **Secrets**。
5. 編輯 **GOOGLE_SERVICE_ACCOUNT_PRIVATE_KEY**，**貼上剛複製的整段**（含雙引號），儲存。
6. 到管理後台點「檢查日曆與寄信設定」→ 若日曆顯示「已設定」，再下一筆測試預約，看日曆是否有新活動。

若 Supabase 不給貼多行，就貼成**一行**：  
把 JSON 裡 `private_key` 的值**去掉前後雙引號**，裡面的 `\n` **保留不要刪**，整段貼成一行。例如：

```
-----BEGIN PRIVATE KEY-----\nMIIEvQIBADAN...很多字...\n-----END PRIVATE KEY-----\n
```

---

## 第 3 步：日曆分享（差在這裡會 403）

- **程式會做的**：用服務帳號的 token 對 **GOOGLE_CALENDAR_ID** 這本日曆呼叫「建立活動」。
- **你還要做**：在 **Google 日曆** 裡，把「這本日曆」**分享給服務帳號的 email**（JSON 裡的 `client_email`），權限：**進行變更的權限**。  
沒分享 = API 回 403，活動建立失敗。

---

## 快速對照：三個 Secrets

| Secret 名稱 | 從哪裡來 | 常錯點 |
|-------------|----------|--------|
| **GOOGLE_CALENDAR_ID** | Google 日曆 → 設定與共用 → 整合日曆 → 日曆 ID | 複製錯日曆、多空格 |
| **GOOGLE_SERVICE_ACCOUNT_EMAIL** | JSON 裡的 `client_email` | 跟 private_key 不是同一份 JSON |
| **GOOGLE_SERVICE_ACCOUNT_PRIVATE_KEY** | JSON 裡的 `private_key` **整段**（含 BEGIN/END） | 貼到 Supabase 時被改格式、截斷、換行被吃掉 → 解 base64 失敗 |

---

## 總結：日曆同步差在哪裡？

- **邏輯與程式**：不差，都已接好（預約 → 呼叫 sync → 用私鑰換 token → 建立日曆活動 → 寫回 event_id）。
- **實際差的那一塊**：  
  1）Supabase 的 **GOOGLE_SERVICE_ACCOUNT_PRIVATE_KEY** 要是「完整、未損壞」的 PEM（照上面方式從 JSON 整段貼上）；  
  2）該本日曆要在 Google 日曆裡**分享給服務帳號**。  
做好這兩點，日曆同步就會過。
