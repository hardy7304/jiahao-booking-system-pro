# 顯示與預約 — 你只需要做這 3 件事

預約**一定會寫入資料庫**（衝突、公休都會擋）；剩下只有「日曆同步」和「寄信」需要你在後台設定。照下面做即可。

---

## 1️⃣ 管理後台先跑一次「檢查」

1. 登入 **管理後台** → **公休與日曆** 分頁  
2. 點 **「檢查日曆與寄信設定」**  
3. 看結果：
   - **Google 日曆：已設定** → 預約會自動同步到日曆，不用再改  
   - **Google 日曆：未設定或私鑰錯誤** → 做下面第 2 步  
   - **預約確認信：已設定** → 會寄確認信，不用再改  
   - **預約確認信：未設定** → 做下面第 3 步  

---

## 2️⃣ 日曆同步（若檢查顯示未設定）

- 到 **Supabase** → **Edge Functions** → **google-calendar-sync** → **Settings** → **Secrets**  
- 確認這三個都有填對：
  - `GOOGLE_CALENDAR_ID`：你的日曆 ID（日曆設定 → 整合日曆可看到）
  - `GOOGLE_SERVICE_ACCOUNT_EMAIL`：服務帳號 JSON 裡的 `client_email`
  - `GOOGLE_SERVICE_ACCOUNT_PRIVATE_KEY`：**整段**從 JSON 的 `private_key` 複製（含 `-----BEGIN...` 和 `-----END...`）  
- 私鑰若一直報錯：打開 JSON 檔，把 `"private_key"` 那整段（含引號）複製貼上到 Secret，儲存後再按一次「檢查日曆與寄信設定」。

**管理後台內嵌日曆顯示 400 時**（此 400 為 **Google 嵌入頁** 回傳，非我們後端）：
1. 到 **Google 日曆** → 該日曆 **設定與共用** → **存取權限** → 勾選「**提供給所有人使用**」。
2. 關閉可能干擾的瀏覽器擴充功能（如 DuckDuckGo、Bitdefender 等）後重新整理。
3. **直接點「在新分頁開啟 Google 日曆」**（頁面最上方綠色按鈕）— 不受 400 影響，可正常檢視與同步。

---

## 3️⃣ 寄信（若檢查顯示未設定）

- 到 **Supabase** → **Edge Functions** → **send-booking-email** → **Settings** → **Secrets**  
- 至少設定：`RESEND_API_KEY`（到 [Resend](https://resend.com) 建立 API Key）  
- 若要寄給客人（非測試）：在 Resend 驗證你的網域，並設定 `RESEND_FROM_EMAIL`（例如 `booking@你的網域.tw`）。

---

## 總結

| 狀況 | 你要做的 |
|------|----------|
| 預約可以下單、但日曆沒事件 | 做 **第 2 步**（Supabase 日曆三個 Secrets） |
| 預約可以下單、但沒收信 | 做 **第 3 步**（Resend API Key + 網域驗證） |
| 管理後台日曆區塊 400 | 日曆設為「提供給所有人」或點「在新分頁開啟日曆」 |

程式端（顯示、衝突、公休、寫入 DB、非同步日曆/寄信）都已接好；剩下就是上面這 3 件事的設定。
