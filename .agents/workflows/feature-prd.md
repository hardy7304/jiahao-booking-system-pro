# /feature-prd

當使用者輸入 /feature-prd 時，請依序執行：

1. 先確認功能背景：
   - 為什麼要做這個功能
   - 目前痛點是什麼
   - 成功指標是什麼

2. 定義需求範圍：
   - In scope
   - Out of scope
   - 使用者角色
   - 主要使用情境

3. 拆解功能需求：
   - 核心流程
   - 欄位需求
   - 權限需求
   - 通知需求
   - 例外情境

4. 列出技術考量：
   - 前端頁面
   - API / function
   - Database schema
   - n8n / automation
   - 第三方整合

5. 補上驗收條件：
   - 功能完成條件
   - 錯誤處理條件
   - 測試條件
   - 上線條件

6. 最後輸出：
   - 功能摘要
   - PRD 結構稿
   - 任務拆解
   - 風險與依賴項
   - 建議開發順序

資訊不足時，先列出待確認問題。
這個適合你在開始做新模組前，先把需求寫成可交付的規格稿。
