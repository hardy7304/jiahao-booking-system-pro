# /ui-spec

當使用者輸入 /ui-spec 時，請依序執行：

1. 先確認功能目標：
   - 使用者要完成什麼事
   - 主要操作角色是誰
   - 使用場景在哪裡

2. 拆解頁面與區塊：
   - 頁面名稱
   - 區塊名稱
   - 欄位與按鈕
   - 狀態顯示
   - 提示訊息

3. 規劃互動流程：
   - 初始狀態
   - 輸入過程
   - 成功狀態
   - 失敗狀態
   - loading / empty state

4. 補充元件需求：
   - table
   - modal
   - form
   - date picker
   - tabs
   - toast
   - badge

5. 說明前後端配合：
   - 哪些欄位來自 API
   - 哪些操作會寫回資料庫
   - 哪些需要權限控管

6. 最後輸出：
   - 頁面規格
   - 元件清單
   - 狀態流程
   - 驗收標準

請盡量用結構化格式，避免模糊描述。
這個很適合你做管理後台、預約頁、Shadcn UI 規格拆解。
